mcc 项目核心功能

学习用的基础版 coding CLI agent，参考成熟 coding agent 的公开行为逐步搭建。

核心使用链路

1 启动 CLI (src/cli.ts)

读取配置文件 ~/.mcc/config.json，初始化环境

2 进入 REPL 循环 (src/repl.ts)

创建 readline 接口，初始化对话历史（包含 system prompt），启动 Tracer

3 用户输入

接收用户输入，添加到对话历史，开始新的 Turn 记录

4 Agent 主循环 (src/query.ts)

调用 LLM API → 检查响应 → 如有工具调用则执行 → 回填结果 → 循环直到模型返回纯文本或达到上限（20次）

  • LLM 调用：使用 chatCompletion 发送请求
  • 工具调用：解析参数 → 验证 schema → 执行工具 → 记录结果
  • 结果回填：以 role:"tool" 消息格式追加到对话历史
5 Trace 记录 (src/trace/)

记录每一步 LLM 调用、工具执行，生成 trace 文件和 HTML 查看器

技术栈

  • 运行环境：Node.js 22+
  • 语言：TypeScript
  • 运行时：tsx
  • 依赖：
    • zod - 参数验证
    • @vscode/ripgrep - 搜索功能
  • 可选工具：poppler-utils（PDF 读取）

配置与使用

创建 ~/.mcc/config.json 配置文件：

{
  "apiKey": "sk-...",
  "baseURL": "https://api.openai.com/v1",
  "model": "gpt-4o"
}

baseURL 可指向任何 OpenAI 兼容端点（DeepSeek / Qwen / Kimi 等）。

工具功能详解

Read 工具 (src/tools/FileReadTool/)

从本地文件系统读取文件内容，支持多种文件格式。

支持的文件类型

文件类型 处理方式
文本文件 直接读取文本内容，支持行号显示、偏移量和行数限制
图片 (PNG, JPG) 转换为 base64 data URI
PDF 使用 pdftoppm 渲染为 JPEG 图片
Jupyter Notebook (.ipynb) 解析为带 cell 标记的文本 + 输出图片

功能特性

  • 支持从指定偏移量开始读取
  • 支持限制读取行数
  • 自动添加行号显示
  • PDF 支持指定页面范围

Write 工具 (src/tools/FileWriteTool/)

向文件系统写入内容。

安全特性：

  • 先读后写校验：文件存在时必须先读取过才能写入
  • 部分视图拒绝：只读取了部分内容时不允许覆写整个文件
  • 过期检测：检查文件是否在读取后被修改
  • 自动创建父目录

Write 工具使用场景

场景 支持情况 说明
创建新文件 ✅ 完全支持 无需先读取，自动创建父目录
完整覆盖已存在文件 ✅ 完全支持 必须先完整读取过该文件
部分修改文件 ❌ 不支持 当前 Write 工具只能整文件覆盖，后续会添加 Edit 工具支持部分修改
基于部分读取覆盖 ❌ 不支持 只读取了文件片段（使用 offset/limit）时不允许整文件覆盖
连续写入同一文件 ✅ 完全支持 写入后会更新读取状态，可连续写入

使用示例：

  • 创建新文件：直接提供完整路径和内容
  • 修改文件：先使用 Read 工具读取，再使用 Write 工具写入新内容
  • 路径要求：必须使用绝对路径，而非相对路径

Edit 工具 (src/tools/FileEditTool/)

精确字符串替换编辑文件，支持卷曲引号归一匹配和保持原有引号风格。

功能特性

  • 支持精确字符串匹配和替换
  • 支持 replace_all 模式批量替换
  • 唯一性校验：old_string 命中多处但 replace_all=false 时拒绝操作
  • 先读后写 + 过期检测
  • 支持创建新文件（old_string 为空时）
  • .ipynb 文件自动转用 NotebookEdit 工具

Grep 工具 (src/tools/GrepTool/)

基于 ripgrep 的强大搜索功能。

输出模式

  • files_with_matches - 只显示匹配的文件路径（默认）
  • content - 显示匹配的具体内容，支持上下文行数
  • count - 显示每个文件的匹配数量

功能特性

  • 支持完整正则表达式语法
  • 支持 glob 模式过滤文件
  • 支持按文件类型过滤（type 参数）
  • 支持 head_limit 限制结果数量
  • 支持多行匹配
  • 支持大小写不敏感搜索

Glob 工具 (src/tools/GlobTool/)

快速文件模式匹配工具，支持标准 glob 语法。

功能特性

  • 支持 **/ 递归目录匹配
  • 支持 * 和 ? 通配符
  • 可选指定搜索目录
  • 默认结果限制 100 条
  • 返回相对路径节省 token

NotebookEdit 工具 (src/tools/NotebookEditTool/)

专门用于编辑 Jupyter Notebook (.ipynb) 文件的工具。

编辑模式

  • replace - 替换指定 cell 内容（默认）
  • insert - 在指定位置后插入新 cell
  • delete - 删除指定 cell

功能特性

  • 支持按 cell ID 或 cell-N 数字索引定位
  • code cell 修改后自动清空 execution_count 和 outputs
  • 先读后写 + 过期检测
  • nbformat >= 4.5 时自动生成 cell id

LSP 工具 (src/tools/LSPTool/)

基于 Language Server Protocol 的代码智能工具。

支持的操作

  • goToDefinition - 跳转到定义
  • findReferences - 查找引用
  • hover - 悬停信息
  • documentSymbol - 文档符号
  • workspaceSymbol - 工作区符号
  • goToImplementation - 跳转实现
  • prepareCallHierarchy - 调用层次
  • incomingCalls - 被谁调用
  • outgoingCalls - 调用谁

功能特性

  • 自动管理文件打开状态
  • 支持文件大小限制（10MB）
  • 位置类结果自动过滤 gitignore 中的文件

链路 Trace 功能

默认开启（可通过 MCC_TRACE=0 或 MCC_TRACE=false 关闭）。

Trace 文件

每轮对话会记录到 ~/.mcc/traces/：

  • trace.ndjson - 真源，append-only，一行一个 turn
  • trace-data.js - 派生视图，把全部 turn 包成 window.__MCC_TRACE__
  • trace-viewer.html - 双击即看的可视化页（启动时自动生成/更新）

记录内容

  • LLM 调用步骤、思考/回答文本
  • finishReason、token 统计
  • 工具调用入参、结果预览、耗时、isError
  • 每轮状态（ok / error / truncated）

项目结构

src/
├── cli.ts            # 入口：读配置 → 启动 REPL
├── config.ts         # 读 ~/.mcc/config.json
├── repl.ts           # readline 循环
├── query.ts          # agent 主循环（对话 → tool call → 回填 → 循环，上限 20）
├── api.ts            # 裸 fetch 调 OpenAI 兼容 chat/completions（非流式）
├── prompts.ts        # system prompt
├── Tool.ts           # 工具抽象（zod schema → OpenAI tools JSON Schema）
├── readFileState.ts  # 读文件状态表（Write/Edit 先读后写/过期检测的数据源）
├── trace/
│   ├── types.ts      # trace 数据模型（Session→Turn→Step 三层）
│   ├── Tracer.ts     # 每轮追加 NDJSON + 重写 trace-data.js + 落 viewer
│   └── viewerHtml.ts # 自包含 viewer 的 HTML 模板
├── tools/
│   ├── FileReadTool/      # Read 工具
│   ├── FileWriteTool/     # Write 工具
│   ├── FileEditTool/      # Edit 工具（精确字符串替换）
│   ├── NotebookEditTool/  # NotebookEdit 工具（.ipynb 专用）
│   ├── GrepTool/          # Grep 工具
│   ├── GlobTool/          # Glob 工具
│   └── LSPTool/           # LSP 工具（代码智能）
└── utils/
    ├── file.ts       # 文件工具函数
    ├── ripgrep.ts    # ripgrep 执行层
    └── glob.ts       # glob 执行层
